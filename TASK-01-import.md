# TASK-01 · 导入：从 B 站把视频批量种进花园

> 这是连载 #1 期的功能。目标：让用户不用一条条手打，能一次把一堆 B 站视频种进花园。

## 背景与约束（重要，先读 CLAUDE.md 的"隐私红线"）

技术上有个绕不开的坎要先认清：**纯前端静态站直接请求 B 站 API 会被 CORS 挡住**，
而且抓"个人收藏夹"还涉及登录态。本项目**绝不碰用户登录凭证**。
所以"自动导入收藏夹"分两步走，本期只做安全、可静态部署的 MVP：

- **本期 MVP（要做）：粘贴导入。** 用户把视频链接/分享文本/BV 号粘进来，前端解析后种下。
  无需登录、无需后端、无 CORS 问题，可直接静态部署，演示也直观。
- **后续（先别做，只在代码里留好扩展点）：** 浏览器扩展读收藏夹页面 DOM 导出列表，
  或带用户授权的公开收藏夹导入。这些等后面的连载期再做。

## 本期具体要实现

### 1. 入口
在"种下"那行旁边或下方加一个**「批量导入」**按钮，点开一个弹层（modal），里面是一个多行
文本框 + 说明 + "导入"按钮。

### 2. 解析逻辑（建议放 `src/lib/parse.ts`，纯函数，方便测）
输入是用户粘贴的多行文本，逐行/逐段解析，尽量从中提取 **BV 号** 和 **标题**。需支持这些常见格式：

- 完整链接：`https://www.bilibili.com/video/BV1xx411c7mD`（可带 `?` 参数，要去掉）
- 纯 BV 号：`BV1xx411c7mD`
- B 站 App 分享文本，形如：`【某某标题-哔哩哔哩】 https://b23.tv/xxxxx`
  - `b23.tv` 短链解析需要请求网络（有 CORS），**本期不解析短链**：
    把方括号【】里的文字当标题，链接原样存进 `link` 字段即可。
- 一行就是一个纯标题（没链接）：直接当成一株草，`link` 留空。

解析产出：`{ title: string; link: string; bvid?: string }[]`。
- 能提取到 BV 号的，`bvid` 填上，`title` 优先用文本里的标题，没有就先用 BV 号占位。
- 完全空白行、纯标点行跳过。

### 3. 数据模型小改
- 给 `Plant` 加可选字段 `bvid?: string`（types.ts）。**保持向后兼容**，老数据没有这个字段也不能崩。
- 导入时**按 bvid 去重**：已在花园里的 bvid 不重复种；没有 bvid 的按 `title` 去重。

### 4. 接入
- 在 `useGarden` 里加一个 `importMany(items)` action，复用现有种下逻辑，统一 `addedAt = nowOf(state)`。
- 导入完成后用现有 toast 反馈，文案给出数量，例如：`导入了 12 株，跳过 3 株重复`。

### 5. 文案（参与了产品气质，按花园主题写，别写"成功/失败"这种系统腔）
- 按钮：`批量导入`
- 弹层标题：`把待看的视频种进来`
- 文本框 placeholder：示例三种格式各一行，让用户照着贴。
- 空输入提示：`先粘点东西进来吧 🌱`

## 验收标准

- [ ] 粘贴混合格式（链接 + BV 号 + 纯标题 + 空行）能正确解析并种下，空行被忽略。
- [ ] 重复的（同 bvid 或同标题）被跳过，toast 如实报出种下/跳过数量。
- [ ] 刷新页面后导入的草还在（localStorage 生效），老用户已有数据不丢。
- [ ] `npm run build` 通过，无 TS 报错。
- [ ] 解析逻辑在 `lib/parse.ts` 里是纯函数，给几个代表性输入手动验证过。
- [ ] 演示模式、枯萎、看完开花等原有功能不受影响。

## 录屏 / 连载提示（做给用户参考，非代码要求）

这期录屏的爽点是："我把我那个吃灰的收藏夹一整页复制粘贴进来，唰一下全变成花园里的草，
其中好几株一进来就是枯的——因为它们躺太久了。"解析时如果遇到奇葩格式 debug，
那段过程也值得留进视频，弹幕爱看。
