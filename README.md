# 🌱 待看花园 · Daikan Garden

> 把 B 站待看清单变成一片会开花、会枯萎的花园。
> 种下想看的视频是 🌱，看完就开花 🌸，拖着不看会慢慢枯萎 🥀。

**「build in bilibili · AI 创造公开赛」参赛作品** —— 公开做产品，观众即用户。

## 这是什么

你的收藏夹是不是也躺着几百个「以后再看」、然后再也没看过的视频？

待看花园把这份愧疚变成一个会生长的小游戏：

- **种下** —— 把想看的视频种进花园，长成一株小草 🌱
- **开花** —— 看完了点一下，小草开花 🌸，还能涨经验
- **枯萎** —— 躺太久没看的草会一天天蔫下去：等待 ≥3 天开始枯萎 🥀，≥7 天就快枯死了
- **起死回生** —— 把一株快枯死的草看完，奖励翻倍（+25 XP），救回来很有成就感

时间拖得越久，花园越荒。看得越勤，花开得越多。

## ✨ 功能

- 🌱 **种下 / 看完 / 移除** —— 一行一株，看完即开花
- 📥 **批量导入** —— 把吃灰的收藏夹整页复制粘贴进来，唰一下全变成花园里的草
  支持 B 站完整链接、纯 BV 号、App 分享文本（`【标题】 https://b23.tv/...`）、纯标题，混贴也能解析，自动按 BV 号 / 标题去重
- 📊 **花园统计** —— 待看、已开花、快枯了一目了然
- 🎮 **等级 & 经验** —— 看完涨 XP，升级
- 💬 **弹幕共建 · 功能许愿池** —— 下期做什么由弹幕投票决定
- ⏩ **演示模式** —— 录屏时一键「快进 3 天」，让草当场枯给观众看

## 🛠 技术栈

- **Vite + React 18 + TypeScript**，纯前端，无后端
- 数据存浏览器 `localStorage`，刷新不丢
- 可 build 成静态站直接部署，方便冲 B 站「Toy」网页内测

## 🚀 本地运行

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # 出静态站到 dist/
```

需要 Node 18+。

## 🔒 隐私红线

**绝不读取、存储或传输你的 B 站登录凭证**（cookie、SESSDATA、密码、token）。
导入一律走「用户主动粘贴公开信息」，不碰登录态、不走服务端代理、无 CORS 烦恼。
